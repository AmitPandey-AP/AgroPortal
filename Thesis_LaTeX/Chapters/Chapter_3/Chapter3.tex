\section{Methodology}

The methodology outlines the step-by-step approach followed to develop the Agricultural Web Portal. The main aim is to create a simple, useful, and reliable system that supports farmers with important information and basic prediction features. The methodology for AGROPORTAL defines the systematic approach used to transform raw environmental data into actionable agricultural intelligence.

%=====================================================%
\subsection{System Architecture}
%=====================================================%

AGROPORTAL is built on the MERN stack with a Python ML inference sidecar. The system follows a multi-layered client-server architecture where modern frontend, backend, database, and machine learning technologies work together to provide intelligent agricultural services. The frontend is developed using React.js with Vite to create a fast, responsive, and component-based user interface. The backend utilises Node.js and Express.js to manage REST APIs, authentication, routing, and communication between different system modules. MongoDB with Mongoose ODM is used as the database layer.

\subsection{Proposed Methodology}
%=====================================================%

The proposed methodology of AGROPORTAL follows a systematic and data-driven approach. The system begins with data collection from publicly available agricultural datasets, soil databases, government weather archives (IMD), and Kaggle datasets. After collection, the data undergoes preprocessing: cleaning, normalisation, handling missing values, and feature selection.

The processed data is then used to train multiple machine learning models including Random Forest, Decision Tree and regression-based algorithms. These models are integrated into the portal through a Python-based inference layer, enabling real-time predictions through REST APIs.

The five primary ML tasks addressed are:
\begin{enumerate}
    \item \textbf{Crop Recommendation} --- What crop should I grow given my soil and climate?
    \item \textbf{Fertilizer Recommendation} --- What fertilizer is optimal for my crop and soil conditions?
    \item \textbf{Regional Crop Prediction} --- Which crops historically thrive in my state/district/season?
    \item \textbf{Yield Prediction} --- How much produce (in Quintals) can I expect from a given area?
    \item \textbf{Rainfall Prediction} --- What is the historical average rainfall for my region this month?
\end{enumerate}

%-----------------------------------------------------%
\subsubsection{Algorithm 1 --- Random Forest Classifier (Crop Recommendation)}
%-----------------------------------------------------%

Random Forest is an ensemble learning method that constructs a multitude of decision trees and outputs the mode of the classes for classification. It uses bagging and random feature selection to build diverse, de-correlated trees.

\paragraph{Mathematical Formula (Majority Vote)}
\begin{equation}
\hat{y} = \arg\max_{c} \sum_{t=1}^{B} \mathbf{I}(\hat{y}_t = c)
\label{eq:rf_majority}
\end{equation}

\paragraph{Entropy and Information Gain (Split Criterion)}
\begin{equation}
H(S) = -\sum_{i=1}^{n} p_i \log_2(p_i)
\label{eq:entropy}
\end{equation}
\begin{equation}
IG(S, A) = H(S) - \sum_{v \in \text{values}(A)} \frac{|S_v|}{|S|} H(S_v)
\label{eq:infogain}
\end{equation}

\subsubsection{Algorithm 2 --- Decision Tree Classifier (Fertilizer Recommendation)}
%-----------------------------------------------------%

A Decision Tree splits the dataset based on the most significant input features. In AGROPORTAL, it is used for fertilizer recommendation with Gini Impurity as the split criterion.

\paragraph{Gini Impurity}
\begin{equation}
\text{Gini}(S) = 1 - \sum_{i=1}^{n} p_i^2
\label{eq:gini}
\end{equation}
\begin{equation}
\text{Gini Gain}(S, A) = \text{Gini}(S) - \sum_{v} \frac{|S_v|}{|S|} \cdot \text{Gini}(S_v)
\label{eq:gini_gain}
\end{equation}

Categorical features (Soil Type and Crop Type) are encoded using \texttt{LabelEncoder} before training.

\subsubsection{Algorithm 3 --- Custom Hand-Built CART Decision Tree (Regional Crop Prediction)}
%-----------------------------------------------------%

The regional crop prediction model uses a hand-implemented CART Decision Tree built entirely from scratch in Python. The key functions are:

\begin{lstlisting}[language=Python, caption={Core CART Functions}, label={lst:cart}]
def gini(Data):
    classes = Data[:, -1]
    n = len(classes)
    _, counts = np.unique(classes, return_counts=True)
    probabilities = counts / n
    return 1 - np.sum(probabilities ** 2)

def info_gain(left, right):
    n = len(left) + len(right)
    parent_gini = gini(np.vstack([left, right]))
    weighted = (len(left)/n)*gini(left) + (len(right)/n)*gini(right)
    return parent_gini - weighted

def build_tree(Data, depth=0):
    if len(np.unique(Data[:, -1])) == 1 or len(Data) < 2:
        return LeafNode(Data)
    split = find_best_split(Data)
    if split is None:
        return LeafNode(Data)
    feature_idx, threshold = split
    left  = Data[Data[:, feature_idx] <= threshold]
    right = Data[Data[:, feature_idx] >  threshold]
    return DecisionNode(feature_idx, threshold,
        left_branch  = build_tree(left,  depth+1),
        right_branch = build_tree(right, depth+1))
\end{lstlisting}

The trained tree is serialised using \texttt{joblib.dump(tree, 'filetest2.pkl')} for runtime inference without retraining.

\subsubsection{Algorithm 4 --- Random Forest Regressor (Yield Prediction)}
%-----------------------------------------------------%

Yield prediction uses Random Forest for regression (predicting a continuous value --- production in Quintals):
\begin{equation}
\hat{y} = \frac{1}{B} \sum_{t=1}^{B} \hat{y}_t(x)
\label{eq:rf_regression}
\end{equation}

Categorical columns are converted using \texttt{OneHotEncoder}. Evaluation metrics include:

\subsubsection{Algorithm 5 --- Historical Mean Lookup (Rainfall Prediction)}
%-----------------------------------------------------%

The rainfall prediction module uses a statistical historical averaging approach using 115 years of IMD data:
\begin{equation}
\text{Rainfall}(\text{SUB}, M) = \frac{1}{N} \sum_{\text{year}=1901}^{2015} \text{Rainfall}(\text{year}, \text{SUB}, M)
\label{eq:rainfall}
\end{equation}

\begin{lstlisting}[language=Python, caption={Rainfall Lookup Implementation}, label={lst:rainfall}]
import pandas as pd, sys
region = sys.argv[1]   # e.g., "KERALA"
month  = sys.argv[2]   # e.g., "JUN"
df     = pd.read_csv('rainfall_in_india_1901-2015.csv')
subset = df[df['SUBDIVISION'].str.upper() == region.upper()]
result = subset[month].mean()
print(round(result, 2))
\end{lstlisting}

\subsection{Implementation}
%=====================================================%

The implementation of AGROPORTAL is divided into three distinct user modules.

\subsubsection{Farmer Module}

The Farmer Module provides a dashboard for all decision-support tools. After secure login via two-factor email OTP authentication, farmers can access:
\begin{itemize}
    \item \textbf{AI Intelligence Hub:} Crop Recommendation, Fertilizer Advisory, Regional Crop Prediction, Yield Estimator, Rainfall Tool, AI Chatbot, Weather Dashboard, Agricultural News
    \item \textbf{Product Management:} Add, edit, and remove crop stock with pricing and images
    \item \textbf{Orders Dashboard:} View and manage incoming customer orders
\end{itemize}

\subsubsection{Customer Module}

This module provides a marketplace for buyers. Features include:
\begin{itemize}
    \item Searchable and filterable product catalogue
    \item Shopping cart with persistent state
    \item Checkout flow with order confirmation
    \item Invoice generation for every successful transaction
    \item Order history dashboard
\end{itemize}

\subsubsection{Admin Module}

The Admin Module provides global oversight:
\begin{itemize}
    \item View and manage all registered farmers and customers
    \item Monitor platform-wide crop availability and stock levels
    \item Respond to user queries via the ``Contact Us'' portal
    \item View system analytics --- total users, total orders, revenue trends
\end{itemize}
